Introduction

What Is Multi-paradigm Programming?

Multi-paradigm programming is programming that enables distinct styles of programming to be implemented whether individually or blended together. Programming languages on the other hand, are categorized according to their paradigms such as imperative, functional and object-oriented. C++ is a multi-paradigm language because programmers are permitted to combine parts of programming approaches such as procedural programming or object-oriented programming.

Advantages & Disadvantages of C++

Advantages:
  • I. Easier compared to other low-level languages
  • II. Supports and allows user-defined operators such as Operator Overloading and function overloading
  • III. Gives programmer complete control over the memory management
    • Disadvantages:
  • I. The concept of pointers used is hard to comprehend
  • II. Have security concerns due to the existence of friend functions, global variables, and pointers
  • III. Lack garbage collector feature to filter out unnecessary data automatically

    C++ By Criteria

    3 Criteria of Programming Paradigms in C++

    Imperative Programming in C++

    C++ is an imperative programming language that supports imperative paradigm. Imperative programming paradigm is one of the oldest programming paradigm and was developed using machine-languages. A program based on this paradigm is made up of a clearly-defined sequence of instructions to a computer. Therefore, the source code for imperative languages such as C++ is a series of commands, which specify the operations that the computer must complete to reach its final goal. The program describes steps that change the computer's state of memory by providing statements. The state is represented by program variables where the values used in variables are changed at program runtime.

    Advantages & Disadvantages of Imperative Programming

    Advantages:
  • I. Very simple to implement
  • II. It contains loop
  • III. Code is easy to understand
    • Disadvantages:
  • I. Complex problem can not be solved
  • II.Higher risk of errors when editing
  • III. Parallel programming is not possible


    • Functional Programming in C++

    Functional progamming paradigm is a programming paradigm where programs are constructed by applying and composing functions. It is a declarative programming paradigm in which emphasize the definition of function that return a value, rather than a sequence of imperative statements which change the state of the program. C++ is not a functional programming language but it is a multi-paradigm programming language that supports functional style programming.

    Functional programming in C++ allows creation of STL algorithm - a set of common algorithm pattern such as sorting and counting which allow to pass custom functions without resorting to function pointers. The capability to pass functions as arguments to another function, and to have functions that return new functions, made even the first standardized version of C++ a functional programming languages. The functional features of the languages are introduced in C++11 standard. Only the C++ compiler which supports the above standard can compile and run the code.

    Advantages & Disadvantages of Functional Programming

    Advantages:
  • I. Program are stateless
  • II. Well-suited for parallelization
  • III. Easily testable and debuggable
    • Disadvantages:
  • I. Data cannot be changed
  • II. Retrieval of large amounts of data not possible efficiently
  • III. Recursive programming can lead to serious errors


    • Object-oriented Programming in C++

    Object-oriented programming is a programming style that uses object to implement the real world entities like inheritance, encapsulation, polymorphism, abstraction and more in programming. The main aim of Object-oriented programming is to bind the data and the functions that operate on them so that no other part of the code can access this data except that function. In C++ languages, the building block that leads to Object-oriented programming is a Class. It is a user-defined data type, which holds its own data members and member functions, which can be accessed and used by creating an object or instance of that class. A Class in C++ is a blue-print representing a group of objects which shares some common properties and behaviours. When a class is defined, no memory is allocated but when it is instantiated where an object is created, memory is allocated.

    Advantages & Disadvantages of Object-oriented Programming

    Advantages:
  • I. Code is reusable
  • II. Ensure data security
  • III. Code is easy to maintain
    • Disadvantages:
  • I. It can cause duplication
  • II. Large program size
  • III. Not suitable for all types of problems
